# Sistemas Operacionais: Guia de Estudo e Arquitetura

Este documento serve como material de apoio para as aulas de **Sistemas Operacionais (SO)**, abordando os conceitos estruturais, gerenciamento de recursos e a evolução dos sistemas de computação.

---

## 1. Conteúdo Programático da Aula

*   **Introdução:** O papel do SO como intermediário entre hardware e software.
*   **Gerenciamento de Processos:** Conceito de processos, threads e escalonamento.
*   **Gerenciamento de Memória:** Memória física, virtual, paginação e segmentação.
*   **Sistemas de Arquivos:** Organização, estruturas de diretórios e alocação de espaço.
*   **Gerenciamento de E/S:** Drivers, interrupções e controle de periféricos.

---

## 2. O que é um Sistema Operacional?

Um **Sistema Operacional** é um software de sistema que gerencia os recursos de hardware do computador e fornece serviços comuns para os programas de aplicativo.

### 🎯 Objetivos Principais
*   **Conveniência:** Tornar o uso do computador mais eficiente e simples para o usuário.
*   **Eficiência:** Permitir que o hardware seja utilizado da melhor maneira possível.
*   **Abstração:** Ocultar a complexidade do hardware real por meio de interfaces padronizadas (System Calls).

---

## 3. Arquitetura do Sistema Operacional

A estrutura interna de um SO define como o núcleo (**Kernel**) interage com o restante do hardware e das aplicações.
